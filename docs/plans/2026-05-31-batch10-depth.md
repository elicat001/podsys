### Task E1 商品库管理(/api/products 增强,新建独立 router 文件,勿改 routers/products.py)
Product 已有字段:title/category/price/print_path/mockup_path/attrs/**sku/batch/source/risk/tags**。
- `routers/product_admin.py` 前缀 `/api/products`(路径错开既有,用 /search、/batch、/{id}/tags),均 current_user + owner 隔离:
  - `GET /search`:query 参数 `source?/risk?/batch?/sku?/tag?/listing_status?`(listing_status:用该商品有无 Listing 及其 status 判断 published/draft/none)+ 分页 `limit=50/offset=0`;返回过滤后的商品列表 + total。
  - `POST /batch`:body `{action:"delete"|"set_risk"|"add_tag", product_ids:[int], value?}`,仅对本人商品生效,返回 affected 数。action 非法 400;product_ids 限长 ≤500。
  - `POST /{id}/tags`:body `{tags:[str]}` 覆盖该商品 tags(非本人 404)。
- 测试:建若干商品(用 /api/products,可在 attrs/字段上设不同 source/risk——若 /api/products 不支持设这些,直接在测试里用 ORM 造或调本 router 不依赖;简单起见:建商品后用 /{id}/tags 设标签,再 /search?tag= 命中)。覆盖:search 按 tag/risk 过滤;batch delete 影响数正确且他人商品不受影响;batch 非法 action 400;tags 设置 + 越权 404;未登录 401。

### Task E2 我的空间深度(/api/space)
Asset 已有字段:owner_id/name/source/risk/created_at/**deleted/batch/tags/size_bytes**。
- `services/quota.py`:`usage(db, owner_id) -> dict`:按 source 分类统计该用户**未删除** Asset 的数量与 `sum(size_bytes)`(采集 collected / 素材 其它),回收站(deleted=True)单独统计;总用量 + 一个演示配额上限(如 settings 没有就常量 2GB)+ 占比 + 是否超容。
- `routers/space.py` 前缀 `/api/space`(均 current_user):
  - `GET /quota` → usage()。
  - `GET /assets`:筛选 `source?/batch?/tagged?(bool)/risk?/q?(名称模糊)`,默认排除 deleted;分页;返回列表 + total。
  - `POST /assets/{id}/trash`(软删 deleted=True,非本人 404)、`POST /assets/{id}/restore`、`GET /trash`(列 deleted)、`DELETE /assets/{id}/purge`(真删行,非本人 404)。
- 测试:传 2~3 个素材(/api/assets);GET /quota 总数/字节正确;trash 后 /assets 不含、/trash 含;restore 回到 /assets;purge 后都不含;filter by source/tagged;越权 404;未登录 401。
